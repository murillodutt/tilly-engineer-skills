Português

Manual de uso

Guia prático para instalar, atualizar e auditar o Tilly Engineer Skills em projetos reais, sem cópia cega de arquivos.

O que é

Tilly instala um mesh de governança para agentes de código. Codex, Claude Code e Cursor passam a apontar para a mesma fonte durável do projeto: docs/agents/**.

Arquivos como AGENTS.md, CLAUDE.md e regras do Cursor devem ser bootloaders finos. A governança real vive em docs/agents/**.

Instalação rápida

  1. Abra o projeto alvo no Codex, Claude Code ou Cursor.
  2. Cole o prompt de instalação do README ou de docs/install/MINI-PROMPT.md.
  3. Deixe o instalador rodar o Step Zero antes de qualquer edição.
  4. Escolha uma rota: current, codex, claude, cursor, all ou audit.
  5. Leia o relatório final antes de pedir commit ou push.

Rotas

RotaQuando usar
currentAtualiza apenas o runtime atual. É a rota padrão.
codexPrepara AGENTS.md e skill Codex.
claudePrepara CLAUDE.md e assets Claude.
cursorPrepara regras em .cursor/rules/**.
allPrepara Codex, Claude e Cursor no mesmo projeto.
auditInspeciona sem modificar arquivos.

Estados do projeto

EstadoSignificado
newSem instruções relevantes de agente. O instalador cria o mesh mínimo.
existingJá existem regras ou docs locais. O instalador faz retrofit, não overwrite.
meshedTilly já existe. A execução vira update/convergence.

Cortex

Cortex é a memória compilada do projeto. A memória vive em Markdown versionado; SQLite é apenas índice derivado.

  CONTRACT.md
  MAP.md
  TRAIL.md
  LINKS.md
  sources/
  cells/
.tilly/cortex/recall.sqlite

Cells precisam ter ## Claim e ## Evidence. sources/** é imutável depois do import. recall.sqlite pode ser apagado e reconstruído.

Obsidian

O Cortex é compatível com Obsidian porque usa Markdown comum. O instalador não cria nem edita .obsidian/**. Obsidian é superfície visual, não fonte de verdade.

Relatório final

Todo fluxo termina com Tilly Context Mesh Convergence Report. O relatório sempre deve expor este manual por link web ou path local.

English

User manual

A practical guide for installing, updating and auditing Tilly Engineer Skills in real projects without blind file copying.

What it is

Tilly installs a governance mesh for coding agents. Codex, Claude Code and Cursor route to the same durable project source: docs/agents/**.

Files such as AGENTS.md, CLAUDE.md and Cursor rules should stay thin bootloaders. Real governance lives in docs/agents/**.

Quick install

  1. Open the target project in Codex, Claude Code or Cursor.
  2. Paste the installer prompt from the README or docs/install/MINI-PROMPT.md.
  3. Let the installer run Step Zero before any edit.
  4. Choose a route: current, codex, claude, cursor, all or audit.
  5. Read the final report before asking for commit or push.

Routes

RouteWhen to use it
currentUpdates only the current runtime. This is the default route.
codexPrepares AGENTS.md and the Codex skill.
claudePrepares CLAUDE.md and Claude assets.
cursorPrepares rules in .cursor/rules/**.
allPrepares Codex, Claude and Cursor in the same project.
auditInspects without modifying files.

Project states

StateMeaning
newNo relevant agent instructions. The installer creates a minimal mesh.
existingLocal rules or docs already exist. The installer retrofits, not overwrites.
meshedTilly already exists. The run becomes update/convergence.

Cortex

Cortex is the project's compiled memory. Memory lives in versioned Markdown; SQLite is only a derived index. 

docs/agents/cortex/
  CONTRACT.md
  MAP.md
  TRAIL.md
  LINKS.md
  sources/
  cells/
.tilly/cortex/recall.sqlite

Cells must include ## Claim and ## Evidence. sources/** is immutable after import. recall.sqlite can be deleted and rebuilt.

Obsidian

Cortex is Obsidian-compatible because it uses plain Markdown. The installer does not create or edit .obsidian/**. Obsidian is a visual surface, not the source of truth.

Final report

Every run ends with a Tilly Context Mesh Convergence Report. The report must always expose this manual through a web link or local path.
