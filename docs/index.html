§ 01

Most agent rules are folklore. Tilly makes them measurable.

A context rule is not project truth until behavioral lift (the agent makes measurably different decisions vs a documented baseline) is proven, the rule survives distractor tasks (typo fixes, README rewordings — it does not pollute lightweight work), ablation loss shows up when the rule is removed, raw evidence stays retained, and a Build–Test–Fail–Fix loop converges. Until then it is taste, prose, or prompt decoration.

Tilly is not copy-paste instructions. It is a five-layer governance mesh connecting narrative → instruction → execution → verification → return — across Codex, Claude Code, and Cursor.

Instructions should not be motivational text. They must become contract, measurement, gate, benchmark, or transferable memory. — PRINCIPLES.md · Evidence-Converged Context

§ 03

Every always-loaded rule pays rent.

Rules consume tokens and attention on every turn. Tilly measures whether each one actually changes a real agent decision — and removes the ones that do not.

loss = 0 · token cost non-trivial →

Remove, condense, or move on-demand.

loss ≥ 2 →

Keep. It changes behavior.

loss = 1 →

Add adversarial eval before deciding.

full fails →

Instruction is ambiguous or insufficient.

distractor leaks →

Context is activating too broadly.

context_mesh_run.py runs two parallel eval sets: trigger evals (the rule must activate on real problems) and distractor evals (the same rule must stay quiet on a typo fix, a formatting change, a read-only lookup). A distractor leak — heavy context activating on light work — is a certification failure, not a warning.

§ 04

A score that zeroes when you cheat.

Adoption is multiplicative, not cumulative. If any factor is zero, the work is not converged.

EffectiveE = Assumptions visibleA × Scope surgicalS × Changes traceableC × Verification passedV

Hover any letter. Each is 0 or 1. One zero collapses the run.

A — assumptions, ambiguity, tradeoffs and blockers were visible before action. S — speculative scope was deleted before implementation. C — every changed line traces to the request. V — a concrete oracle passed before closure. None of these is rhetorical: each maps to a check in docs/mesh/PRINCIPLES.md.

For projects with persistent context — skills, hooks, MCPs, durable docs — there is a second formula: S = B · R · M · D · G · O · C. Bootloader stays short. Routing selects the smallest correct source set. Mode boundaries — instruction, context, user input, validation — are explicit (M). The four-gate Discipline is available before implementation. Tools, hooks, plugins, MCPs and subagents are Governed before use. Checks are wired into local development or CI (Oracle integration). A new window can recover state and the next check (Continuity).

§ 07

Cortex — memory that survives the chat.

Markdown filesystem is truth. SQLite FTS5 is derived recall. rg is fallback. Cortex compiles project memory into versioned, auditable, Obsidian-visible Markdown — every claim grounded by a link to its source. 

docs/agents/cortex/
  CONTRACT.md          // local Cortex operating contract
  MAP.md               // navigable catalog
  TRAIL.md             // append-only chronological record
  LINKS.md             // plain-language relationship map
  sources/             immutable
  cells/               claim + evidence

.tes/cortex/
  recall.sqlite        // derived FTS5 index, never source
  semantic.sqlite      // derived semantic curation cache

Cells must include ## Claim and ## Evidence. sources/** is immutable after import. The MCP layer is read-only — seven tools, zero writes:

cortex_verify

Validates required files, directories, and contract terms.

cortex_audit

Detects broken links, missing evidence, unlisted cells, orphans.

cortex_recall

SQLite FTS5 search; falls back to rg.

cortex_read_cell

Reads one file under cells/** — direct, no caching.

cortex_absorb_plan

Generates a no-write plan to import a source.

cortex_curate_plan

Classifies semantic risks — merges, splits, tensions, evidence gaps — without writing memory or derived indexes.

cortex_reflect

Generates a no-write closure and curation proposal.

Backends are explicit about failure: --backend lexical is the deterministic gate. --backend xenova requires @huggingface/transformers and reports BLOCKED when unavailable. --backend auto reports DEGRADED when it falls back. No silent fallbacks. No hidden quality drops.

§ 08

Privacy as a contract.

Field Reports lets Tilly learn from real installations — without surveillance. Two privacy gates: local sanitization on capture, GitHub-side schema enforcement on receipt. If the GitHub CLI, network, or auth are unavailable, the outbox stays pending and your push proceeds unchanged.

Never sends

Eleven things Field Reports refuses to transmit.

  • Code
  • Diffs
  • Prompts
  • File contents
  • Raw stack traces
  • Secrets & tokens
  • Personal data
  • Absolute paths
  • Raw branch names
  • Raw remote URLs
  • Tables & code blocks

Reports are factual prose and bullets. Identity is a random per-install install_id — published as a hash fingerprint, not the raw value. The receiver workflow rejects any report containing forbidden patterns, quarantines the rest. Opt out with one line typed in the agent window: 

/tes:field-reports:disable
§ 09

Fourteen deterministic gates. No opinions.

Ten oracles plus four validators sit between intent and closure. Each one fails repeatably or passes with retained evidence. No "looks good" declarations.

project_context_oracle.py

Validates PROJECT-CONTEXT.md has all 17 required sections with real content — not lorem ipsum.

project_alignment_oracle.py

Checks ROADMAP uses controlled terms, BOUNDARIES declares categories, GATES use gate-term vocabulary.

command_trigger_oracle.py

Verifies Codex, Claude, and Cursor expose the same /tes-* trigger vocabulary.

adapter_parity_readiness.py

Cross-adapter behavior convergence — proves the four gates fire identically on all three runtimes.

context_mesh_run.py

Runs evals, classifies failures, measures distractor leak rate — heavy context activating on lightweight tasks.

cortex_quality_oracle.py

Validates Cortex structure, cell integrity, and broken-link detection.

field_reports_github_oracle.py

Receives, validates, quarantines GitHub issues against the tes-field-report@2 schema.

platform_surface_oracle.py

Tracks official documentation URLs for GitHub, Codex, Claude, and Cursor — fails when references go stale.

retention_metadata.py

Stamps every benchmark with run_id, run_head, gate_head, retention_head — so evidence is permanent and judgment about its maturity is separate.

§ 11

What is certified — and what is not.

Honest claims are the contract. Tilly publishes certified surfaces with the same vocabulary the agent uses internally. No tag, release, package publish, marketplace submission, or live GitHub issue creation is claimed by this page.

SurfaceStatusBoundary
Assisted install Certified Remote replay and local helper contract checks pass end-to-end.
Cortex MCP Certified · read-only No write-capable MCP tools are exposed in this version.
Obsidian-ready mesh Certified Markdown properties and wikilinks generated. Plugins and .obsidian/** writes are never claimed.
Field Reports Partial Local capture, fake transport, receiver quarantine certified. Live publication requires explicit authorization.
Marketplace · release actions Not taken No tag, release, package publish, or marketplace submission has been performed.
Vector / Xenova backend Deferred Optional dependency; not installed or certified by default.
